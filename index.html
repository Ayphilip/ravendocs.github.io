Getting started

Raven is a decentralized Web3 social application built as an X (Twitter) clone, leveraging blockchain technology for secure data storage. Unlike traditional social media platforms, Raven ensures data integrity, immutability, and enhanced security by storing posts, interactions, and user details on the Celestia blockchain. The application is developed using the Chopin Framework, allowing seamless Web3 integration.

Introduction

Raven is designed to empower users with full control over their data while offering a familiar and intuitive microblogging experience. Key features include:
Decentralized Storage: All tweets and interactions are stored securely on the Celestia blockchain.
Immutable Content: No central authority can alter or delete posts once published.
Web3 Authentication: Users sign in using blockchain-based authentication instead of traditional credentials.
Peer-to-Peer Interactions: Enables decentralized conversations and encrypted messaging.

Setup

To setup Raven, follow these steps.

Prerequisites
  • Node.js (LTS version recommended)
  • Chopin CLI (npx chopd)
  • Firebase Configuration
  • ReactJs Vite
Installation
  1. Clone the Raven repository: 
    git clone https://github.com/Ayphilip/raven.git
    cd raven
  2. Install Dependencies for Backend: 
    yarn install
# or
npm install
  3. Install Dependencies for Frontend: 
    cd frontend
npm install
  4. Initialize Chopin Framework on root folder: 
    npx chopd init
  5. Run Application:
    on backend 
    npx chopd init
    on frontend 
    cd frontend
npm run dev
  6. Open Application:
    - Backend: http://localhost:4000/
    - Frontend: http://localhost:5173/
Firebase Setup

Follow the [Firebase Documentation] to initialize and create a new Firestore database.

Environment Variables: Create a .env file in the root directory and add the following:

  1. Clone the Raven repository: 
    FIREBASE_API_KEY=your_api_key
FIREBASE_AUTH_DOMAIN=your_project_id.firebaseapp.com
FIREBASE_PROJECT_ID=your_project_id
FIREBASE_STORAGE_BUCKET=your_project_id.appspot.com
FIREBASE_MESSAGING_SENDER_ID=your_sender_id
FIREBASE_APP_ID=your_app_id
FIREBASE_MEASUREMENT_ID=your_measurement_id
Deployment
  1. Clone the Raven repository: 
    git clone https://github.com/Ayphilip/raven.git
    cd raven
  2. Install Dependencies and build application: 
    npm run build
  3. Start Application:
    on Root folder 
    npx chopd
  4. Open Application:
    - Application: http://localhost:4000/

Chopin Framework

Raven leverages the Chopin Framework to seamlessly integrate blockchain security while maintaining the familiar web development experience. Chopin transforms Raven into a sovereign rollup and embeds a Multi-Party Computation (MPC) wallet directly in the browser, ensuring a frictionless user experience.

Implementing Chopin Framework, Raven achieves decentralized security while maintaining a user-friendly, accessible, and scalable design.

npx chopd – CLI Tool for Chopin Framework

chopd is a command-line interface (CLI) tool designed to streamline development and testing for applications built with the Chopin Framework. It provides a reverse proxy with enhanced request handling, identity simulation, and WebSocket pass-through.

Getting Started with npx chopd

To initialize an application with the Chopin Framework, run the following command:

npx chopd init

This command initializes the application, creates a Chopin configuration JSON file, and sets up the reverse proxy to the desired port.

Chopin Configuration (chopin.config.json)

After running npx chopd init, a configuration file is generated with the following structure:

{
    "command": "npm start",
    "proxyPort": 4000,
    "targetPort": 5000
}
Configuration Parameters
  • command: The CLI command to start the backend deployment (e.g., in this case "npm start").
  • proxyPort: The port where the backend will be accessible through the proxy.
  • targetPort: The actual backend deployment port. chopd will forward requests from the proxy port to this port.

Using Chopin Oracle in Raven

When initializing an Oracle in Raven, an instance of the Oracle is created with a redirect URL passed into the initialization request. To transmit data on-chain, Chopin intercepts all POST, PUT, UPDATE, and PATCH requests. The backend then reads the x-callback-url header, which is injected by Chopin’s reverse proxy.

app.post("/hello", (req, res) => {
    const callbackUrl = req.headers["x-callback-url"];
    const oracle = new Oracle(callbackUrl);
});

The Chopin Oracle is integrated into the backend to bring non-deterministic data into Raven. This allows the application to process:

  • Real-time external data from third-party APIs and websites. using 
    await oracle.fetch(https://example.com)
  • Current timestamps for event tracking. using 
    await oracle.now()
  • Random numbers for cryptographic security, polling, and other dynamic features. 
    await oracle.random()
  • Non-Deterministic Value It can also be usedd to return arbitrary non-deterministic value using 
    await oracle.notarize(() => )

    For exapmle in Raven Backend implementation, The notarize function was used to return a UUID for every new entry into the database. (All Oracle implementation in Raven is done in the util.js file and then referenced across)

    export const genId = async (req) => {
    const callbackUrl = req.headers["x-callback-url"];
    var oracle = new Oracle(callbackUrl);
    var Id = await oracle.notarize(() => crypto.randomUUID())
    return Id;
};
How Chopin Oracle Works as mentioned by https://chopin.sh
  1. Listening for Context

    The sequencer listens for "context" from the request handler while executing a request. If context is reported, it is added on-chain in the order it was received.

  2. Processing External Data

    The backend reads the x-callback-url header injected by Chopin’s reverse proxy. This header contains the URL of the reporting server that will send data to the sequencer.

  3. Reporting Non-Deterministic Data

    When Raven’s backend generates external or random data, it submits a POST request to: 

    /_chopin/report-context

    The request body contains the value to be stored and retrieved later. When a request is re-executed, Raven fetches the recorded value instead of regenerating it.

Features

User Features

Authentication

Raven ensures secure user authentication through the Chopin Framework, enabling decentralized identity verification. Users can securely log in and manage their accounts while leveraging blockchain-based authentication for increased security and trust.

Profile Management

Users can create and manage their profiles, including setting up avatars, bios, and other details. Profile customization enhances user experience and engagement within the Raven ecosystem.

Post Management

Users can create, edit, delete, and retrieve tweets. Each post is securely stored on the Celestia blockchain, ensuring tamper-proof and immutable content storage.

Comment System

Raven supports a nested reply system, allowing users to engage in dynamic discussions under posts. Each comment is stored securely on the blockchain, maintaining data integrity.

Likes and Repost

Quote Tweets

Users can quote tweets, adding their thoughts and context while referencing original posts, further enriching the discussion within the Raven platform.

Chat and Messaging

Raven provides a decentralized chat system with end-to-end encryption (E2EE), ensuring private and secure messaging between users. All chat-related data is stored on-chain, enhancing security and reliability.

Treasure Hunt

The Treasure Hunt feature is designed to increase engagement by allowing users to participate in gamified quests with rewards. Only verified Raven users can create hunts, while any user can participate.

Treasure Hunt Types

  • Cumulative Reward Hunt: Users pay an entry fee (set by the creator) in Raven Trust Token (RTT), and the first to complete the quest wins the entire pooled reward.
  • Fixed Reward Hunt: The creator sets a fixed reward in RTT, and participants join for free. The first user to solve the quest wins the pre-determined reward.

Token and NFT System

Raven Trust Token (RTT)

RTT serves as the native utility token within the Raven ecosystem, facilitating multiple use cases.

  • Minting Verification Badges: Users pay an entry fee (set by the creator) in Raven Trust Token (RTT), and the first to complete the quest wins the entire pooled reward.
  • Treasure Hunt Participation: The creator sets a fixed reward in RTT, and participants join for free. The first user to solve the quest wins the pre-determined reward.
  • Reward Payment: In Fixed Reward Hunts and Cumulative Reward Hunts, RTT is used to distribute rewards
  • Chat Gifting System: Users can send RTT as gifts within the chat feature, promoting user interactions and incentivizing engagement
